Industrial edge gateways collect packet statistics, protocol traces, sensor activity, and system events that can reveal sensitive operational behavior. Collaborative learning is therefore useful: a single gateway may only observe a narrow subset of normal behavior and attacks, while a fleet can expose a broader threat distribution. Directly centralizing raw telemetry is often unacceptable, and plain federated learning does not by itself protect against a server that inspects individual client updates.

Small language models create a useful interface for edge security assistance: they can classify alerts, support operator-facing explanations, and later connect detection with remediation or policy generation. Yet adapting such models in an IIoT setting introduces three practical constraints. First, edge or near-edge training must keep trainable parameters and memory small. Second, federated communication must be limited because gateway links can be constrained. Third, privacy must be auditable under a clearly stated unit of protection. These constraints are especially demanding under client-side release DP, where each client privatizes its upload before the server receives it.

We therefore instantiate the main system with Qwen3.5-2B~\citep{qwen35modelcard}, a recent open-weight model in the 0.5--3B edge-relevant range, rather than a larger small-language-model backbone. This choice makes the edge-intelligence claim concrete: the paper studies how to privately federate an actually compact 2B assistant. Although Qwen3.5-2B supports multimodal usage, our benchmark uses only the text/language pathway with telemetry rendered as compact textual prompts.

Parameter-efficient fine-tuning methods such as LoRA~\citep{hu2021lora} reduce the number of trainable parameters, but federated averaging of adapter factors can be suboptimal because the product parameterization is not uniquely identified. Recent fixed-basis and square-core approaches, including Fed-SB~\citep{singhal2025fedsb}, directly address this issue by training a small matrix between fixed adapter factors. \method does not claim this square-core idea as new. Instead, it asks a different question: can public spectral structure from the released backbone be used to recover utility under strict client-side DP while respecting edge communication and evaluation requirements for IIoT intrusion detection?

\method answers this question with a public-spectral, budget-aware federated core adaptation design. For each selected linear layer of a frozen small language model, \method derives orthonormal bases from the public backbone weights, trains only a small layer-wise core matrix, clips and noises client uploads in core coordinates, and aggregates the privatized updates on the server. The method is designed around four principles: private-data-free basis construction, public spectrum-driven layer allocation, client-level DP accounting, and local residual personalization for non-IID gateways.

The paper makes the following contributions:
\begin{itemize}[leftmargin=*]
  \item We formulate a client-level DP federated adaptation problem for edge IIoT security assistants, emphasizing privacy, communication, rare attack recall, and reproducible evaluation rather than only aggregate accuracy.
  \item We introduce a public-spectral core adapter that constructs fixed bases from released backbone weights and trains only small core matrices, giving unique coordinates and exact aggregation in the chosen public subspace.
  \item We propose a budget-aware utility recovery strategy for client-side release DP, including public spectrum-driven rank allocation, layer-adaptive clipping/noise allocation, local residual cores, and server-side post-processing.
  \item We define a reproducible \dataset protocol for instruction-style intrusion detection with fixed splits, simulated IID and Dirichlet non-IID client partitions, leakage diagnostics, raw-file robustness checks, and fair comparisons under matched privacy, communication, and parameter budgets.
\end{itemize}

This framing deliberately avoids claiming that a square core or exact core averaging is unique. The intended novelty is the combination of public-backbone spectral construction, DP utility recovery, edge-budget allocation, and an IIoT-specific evaluation protocol.
